## 目的
CEO油谷憲人の意思決定を支援するAI参謀団。  
事業横断の課題整理、思考支援、戦略から実行計画まで落とし、再現性のある経営を設計する。

## 構造

| フォルダ | 役割 |
|----------|------|
| **ai-office** | AI役員（橘・真田・水野・相沢・工藤・神谷）の定義と経営会議モード |
| **company-memory** | 会社の知識ベース。AIが「会社を分かった上で答える」ための長期記憶 |
| **.cursor/rules** | AIの行動ルール（社長室人格・メモリ参照・思考フレーム） |

## 運用

- **週次MTG** … ログは `company-memory/09_meetings/weekly/` に保存。1ファイル1回。
- **月次レビュー** … `company-memory/09_meetings/monthly/` に保存。
- **AI経営会議** … 議題を投げて参謀団で議論。結論と実行計画まで出す。
- **CEOダッシュボード更新** … `company-memory/10_dashboard/` をAIが要約・更新する基準とする。表示はNotion前提。

## 基本コマンド

| 入力 | 動作 |
|------|------|
| **おはよう** | 橘が朝のブリーフィング |
| **この問題を分析して** | Insight → Recommendation → Action plan で分析 |
| **経営会議** または **経営会議 議題：◯◯** | AI経営会議モード（論点→データ→戦略→リスク→仕組み化→実行計画→結論） |
| **厳しめに見て** | 神谷中心のリスク分析強化 |
| **実行計画に落として** | 相沢中心でやること・担当・期限を表形式で出力 |
| **深掘りして** | 複数参謀の視点で論点を深掘り |
| **追加分析** | 水野中心のデータ・分析の追加 |

## 社長の毎日の使い方

### 朝
**「おはよう」**

AIが出すもの：
- 今日の優先事項
- 今週の最大論点
- 今日判断すべきこと

### 昼
**「この問題を分析して」**（論点を書く）

洞察 → 推奨 → 実行計画の形で返す。

### 夜
**「経営会議」** または **「経営会議 議題：◯◯」**

議題に沿って参謀団が順番に議論し、結論と実行計画まで出す。

## CEOダッシュボードとの関係

- **表示** … Notion にCEOダッシュボードを置く（見た目・共有用）。
- **思考・要約** … Claude（AI社長室）が分析・要約する。
- **元データ** … `company-memory/10_dashboard/`（ceo_dashboard.md, weekly_summary.md, alerts.md）をAIが読む・更新する。

**半自動フロー（推奨）**
1. Notion / MTGログを更新  
2. Claudeに要約依頼  
3. CEOダッシュボード更新案が出る  
4. 必要ならNotionに貼る  

完全自動化より、まずはこの半自動で運用するのが扱いやすい。

## 重要な原則

AI社長室は次の役割分担で運用する。

- **AI** = 思考（整理・分析・案の提示）
- **CEO** = 判断（意思決定）
- **役員** = 実行（現場での実施）
