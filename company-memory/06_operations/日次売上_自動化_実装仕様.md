# 日次売上レポート 完全自動化 実装仕様

## 現状の課題

毎日スタッフが以下をブラウザでログインし、**手入力**で Google スプレッドシート（日次売上報告）に入力している。

- Uber（Uber Eats 売上）
- エアレジ（店内売上・客数など）
- LINE 登録者数

→ **これを完全に自動化し、社員の手入力なしでスプレッドシートに反映する。**

---

## 結論：API で自動化可能

| データソース | 自動化手段 | 備考 |
|--------------|------------|------|
| **エアレジ** | データ連携API | バックオフィス「設定」→「他システム連携［データ連携API］」で API キー・トークン発行。[FAQ](https://faq.airregi.jp/hc/ja/articles/48202752451353) |
| **Uber Eats** | Reporting API | Uber Developers の Eats Reporting API。OAuth 要。書面承認が必要な場合あり。[Uber Eats Reporting](https://developer.uber.com/docs/eats/guides/reporting) |
| **LINE 登録者数** | Messaging API（統計） | `GET /v2/bot/insight/followers?date=YYYYMMDD` で友だち数取得。[Messaging API 統計](https://dev.classmethod.jp/articles/update-line-messaging-api-0708/) |
| **Google スプレッドシート** | Google Sheets API | 取得したデータをセルに書き込み。サービスアカウント or OAuth。 |

**ブラウザの手入力は不要。** 1本のスクリプトを「毎日決まった時刻」に実行するだけで、3ソースのデータを取得し、スプレッドシートに書き込む形にできる。

---

## 全体フロー

```
[毎日 指定時刻]
  ↓
スクリプト実行（社員PC or サーバー）
  ↓
① エアレジ API で 店内売上・客数 取得
② Uber Eats API で Uber 売上・件数 取得
③ LINE Messaging API で 友だち数 取得
  ↓
④ Google Sheets API で 日次売上報告シート に書き込み
  ↓
手入力ゼロでスプレッドシート更新完了
```

---

## 実装形態（Cowork や社員 PC を視野に入れる）

- **実行場所**：社員の PC でも可。その場合は **タスクスケジューラ（Windows）** または **cron（Mac）** で「毎日 〇時」にスクリプトを実行する。
- **Cowork**：RPA ツール等で「同じ PC で毎日実行」する場合も、このスクリプトを 1 回叩くジョブとして組み込める。ブラウザ操作ではなく **API 呼び出し＋Sheets 書き込み** にしておけば、安定して動く。
- **推奨**：最初は 1 台の PC（担当者 or 事務）でスクリプトを定期実行。慣れたらサーバーやクラウド関数に移行してもよい。

---

## 必要な認証情報（すべて Notion 等で管理し、スクリプトでは環境変数で参照）

| 種別 | 内容 | 取得方法 |
|------|------|----------|
| エアレジ | API キー、API トークン | バックオフィス「設定」→「他システム連携［データ連携API］」で発行 |
| Uber Eats | OAuth クライアント ID / シークレット、または トークン | [Uber Developers](https://developer.uber.com/) でアプリ作成・Eats スコープ取得。要承認の場合あり |
| LINE | Channel Access Token（長期） | LINE Developers コンソールで Messaging API のチャネル作成 → トークン発行 |
| Google Sheets | サービスアカウント JSON キー | Google Cloud Console でプロジェクト作成 → Sheets API 有効化 → サービスアカウント作成 → キーを JSON でダウンロード。対象スプレッドシートを「編集可」で共有 |

**重要**：これらは **.env ファイル** または OS の環境変数にのみ入れ、リポジトリにはコミットしない。

## フォルダ・ファイル構成（プロジェクト内）

```
AI社長室/
├ scripts/
│  ├ daily_sales_to_sheets/     # 日次売上 自動投入スクリプト
│  │  ├ .env.example            # 必要な環境変数の例（値は空）
│  │  ├ requirements.txt        # Python 依存
│  │  ├ config.py               # 設定・シートID・範囲
│  │  ├ fetch_airregi.py        # エアレジ API 取得
│  │  ├ fetch_uber.py           # Uber Eats API 取得
│  │  ├ fetch_line.py           # LINE 友だち数取得
│  │  ├ write_sheets.py         # Google Sheets 書き込み
│  │  └ main.py                 # 上記を順に実行
│  └ README_automation.md       # セットアップ・実行手順
```

- **main.py** を「毎日 1 回」実行するようにスケジュールする。
- スプレッドシートの「どのシート・どの行に何を書くか」は **config.py** と **write_sheets.py** で指定する（現在の日次売上報告の列構成に合わせる）。

## データのマッピング（スプレッドシート列との対応）

現在の「日次売上報告」シートの列に合わせて、取得した値を次のように割り当てる。

- **日付**：実行日の前日（または「今日」）を想定
- **担当者**：固定 or 未入力のまま（必要なら別ロジックで設定）
- **店内 11–15 時 / 17–23 時 / 23–29 時**：エアレジ API の取引・時間帯集計から算出
- **Uber 11–15 時 / 17–23 時 / 23–29 時**：Uber Reporting API のデータから時間帯別に集計
- **店内平均客単価・Uber 平均客単価・総客数・総売上**：上記から計算
- **LINE 増加数**：LINE API の前日比（または当日の友だち数）を「LINE増加数」列に書き込み

※ エアレジ・Uber の API の実際のレスポンス項目に合わせて、**fetch_airregi.py / fetch_uber.py** 内で集計ロジックを調整する。

## 実行スケジュール例（社員 PC）

- **Windows**：タスクスケジューラで「毎日 朝 6:00」に `python C:\...\main.py` を実行（前日分が確定している時間帯）。
- **Mac**：cron で `0 6 * * * cd /path/to/scripts/daily_sales_to_sheets && python main.py`。
- **Cowork 等**：同じコマンドを「日次ジョブ」として 1 回実行するだけにすると、運用が簡単。

---

## 段階的な導入（推奨）

1. **Phase 1**：**エアレジ API ＋ Google Sheets 書き込み** だけ実装。店内データのみ自動化。手入力は Uber・LINE のみ残す。
2. **Phase 2**：**LINE API** を追加。LINE 登録者数（および増加数）を自動投入。
3. **Phase 3**：**Uber Eats Reporting API** を申請・取得し、Uber 分も自動化して完全自動にする。

これで「毎日スタッフがブラウザでログインして手入力」する作業はゼロにできる。Cowork を社員 PC に入れる場合は、このスクリプトを 1 日 1 回実行するタスクとして組み込む形で実装すればよい。
