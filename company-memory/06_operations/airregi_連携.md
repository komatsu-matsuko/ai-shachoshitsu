# エアレジ連携（AI社長室でデータを参照する仕組み）

## 目的

エアレジのデータをAI社長室が参照し、**経営チーム（AI参謀団）で分解 → 対策 → 社員への指示** まで一気通貫で出す流れにする。

---

## ログイン情報について

- **エアレジのログインID・パスワード** は Notion で管理している（社内で共有しているページを参照）。
- このプロジェクト内には **認証情報を書かない**。スクリプトで使う場合は環境変数（`.env`）で渡し、`.env` は git に含めない。
- ログイン手順だけを書く場合：Notion でID・パスワードを確認 → エアレジのログイン画面で入力。手順の詳細は Notion 側に記載。

※「ログイン方法が分かりましたか？」への回答：**実際のID・パスワードは Notion にあるため、AIは知らないし持たない。** 手順（どこで何を入力するか）は Notion に書いておけば、人がログインしてデータ取得やAPI設定ができる。

---

## 重要な前提：AIはブラウザを操作できない

- **Cursor 上の AI は、ユーザーのブラウザを直接操作したり、エアレジにログインしたりすることはできません。** 拡張機能を使っても、「ブラウザを動かす主体」はユーザー側（または拡張機能）であり、AIがブラウザ内を“見る”ことは現状の仕様ではできません。
- その代わり、**「エアレジのデータをどこか一か所（ファイル）に置く」** と、AIはそのファイルを読んで **分解・対策・社員への指示** まで組みます。

---

## 実現したい流れ（イメージ）

```
エアレジのデータを取得
  ↓
プロジェクト内の「決まったファイル」に保存
  ↓
社長が「エアレジのデータを分析して、対策と社員への指示まで出して」と依頼
  ↓
AIがそのファイルを読み、経営チーム形式で
  ・水野：データ分解・KPI整理
  ・真田：対策・戦略案
  ・相沢：社員への指示（タスク・担当・期限）
  を出力
```

---

## データの取り込み方（3つの選択肢）

### 1. エアレジ API で取得（推奨・半自動）

- エアレジには **データ連携API** がある（[APIで各種システムとデータ連携](https://faq.airregi.jp/hc/ja/articles/48202752451353)）。
- バックオフィス「設定」→「他システム連携［データ連携API］」で API キー・API トークンを発行。
- **API キー・トークンは Notion に保管し、スクリプトでは環境変数で参照。**
- 自作の小さなスクリプト（例：Python / Node）で「取引・売上など」を取得 → 所定のフォルダに **1ファイル** で保存（例：`company-memory/10_dashboard/airregi_snapshot.md` または `04_kpi/restaurant_airregi.md`）。
- ユーザーが「週1回実行」「手動で実行」するだけにすれば、AIはそのファイルを読んで経営チーム形式で分解・対策・指示を出せる。
- **メリット：** 同じ手順で再現しやすく、取得項目を増やしやすい。  
- **デメリット：** API の利用設定とスクリプトの初回準備が必要。

### 2. 手動エクスポート

- エアレジの画面から CSV などでエクスポート → その内容を **同じファイル**（例：`10_dashboard/airregi_snapshot.md`）にコピー＆ペースト、またはファイルをそのフォルダに置く。
- 社長が「このデータを分析して対策と指示を出して」と依頼すると、AIがそのファイルを読んで経営チームで分解・対策・社員への指示を出す。
- **メリット：** 準備不要で今すぐできる。  
- **デメリット：** 都度、人がエクスポート＆配置する必要がある。

### 3. ブラウザ拡張機能（将来的な拡張）

- エアレジの画面を開いた状態で、**拡張機能**が画面上の数値・テーブルを取得し、ローカルの「決まったファイル」に保存する方式。
- この場合も **ブラウザを操作するのは拡張機能（またはユーザー）** であり、**AI が直接ブラウザを操作するわけではない**。  
  「拡張機能がデータをファイルに書き出す → AI はそのファイルを読む」という役割分担になる。
- **メリット：** ログイン済みの画面から自動でデータを取り出せる可能性。  
- **デメリット：** 拡張機能の開発・保守が必要。エアレジの画面変更で動かなくなるリスクあり。

## データを置く場所（AIが参照する）

- エアレジ由来のデータは、次のいずれかに **1ファイル** でまとめておく想定。
  - `company-memory/10_dashboard/airregi_snapshot.md`  
    … ダッシュボード用のスナップショット（売上・客数・日別など）
  - または `company-memory/04_kpi/restaurant_airregi.md`  
    … 飲食KPI用として、エアレジの数値だけをまとめたファイル
- ファイル名とパスを決めたら、**「エアレジの最新データはここを見る」** と運用で統一する。AIはそのパスを読んで分解・対策・指示を出す。

---

## 次のアクション

1. **すぐやりたい場合**  
   - 手動でエアレジからエクスポート（または画面の数値をコピー）→ `company-memory/10_dashboard/airregi_snapshot.md` に貼り付ける。  
   - 社長が「このデータを経営チームで分解して、対策と社員への指示を出して」と依頼する。

2. **半自動にしたい場合**  
   - エアレジの API を有効化し、API キー・トークンを Notion に保管。  
   - 環境変数でキー・トークンを渡すスクリプトを作成し、取得結果を `10_dashboard/airregi_snapshot.md`（または上記の KPI 用ファイル）に保存。  
   - 実行は週次や日次で手動 or タスクスケジューラで実行。

3. **拡張機能でブラウザから取りたい場合**  
   - 「エアレジのこの画面のこのテーブルを、このフォルダのこのファイルに保存する」という仕様を決め、ブラウザ拡張で実装。  
   - 保存先は上記の `airregi_snapshot.md` などに統一すると、同じ「経営チームで分解・対策・指示」の流れが使える。

---

## まとめ

- **ログイン方法：** 手順・ID・パスワードは Notion で管理。AIはブラウザを操作しないので、ログインは人が行う（またはAPI用のトークンでスクリプトが連携する）。
- **「あなたが直接ブラウザを操作し」：** 現状の Cursor では不可。代わりに **「データをファイルに落とす → AI がそのファイルを読んで経営チームで分解・対策・社員への指示を出す」** 流れなら実現可能。
- 拡張機能は「**ブラウザ内のデータをファイルに書き出す**」役割。**AI がブラウザを操作するのではなく、データがプロジェクト内に用意されていることが前提**になる。
