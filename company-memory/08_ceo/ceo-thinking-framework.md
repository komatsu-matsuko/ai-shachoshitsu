# CEO思考フレームワーク

Owner: 油谷憲人

## 思考の型

- **構造化**
- **シンプル**
- **実行可能**
- **再現性**

## 意思決定の流れ

```
仮説
  ↓
テスト
  ↓
フィードバック
  ↓
スケール
```

## CEOの役割（時間設計の目安）

- 戦略 40%
- 外部 30%
- 人材 20%
- 管理 10%

CEOは運営管理を行わない。戦略・判断・人材・外部に集中する。

---

## 意思決定の型（大きな判断の順番）

1. **まず数字を見る**
   - 売上・粗利・費用対効果・再現した場合の規模
2. **再現性があるかを考える**
   - 一度だけの成功か / 仕組み化できるか / 他の現場でも成立するか
3. **小さく試せるかを考える**
   - まずテストできるか / 被害が小さい状態で試せるか
4. **組織として回るかを見る**
   - 誰がやるのか / 人材レベルで成立するか / 属人化しないか
5. **実行してデータを取る**

---

## 判断の切り口（迷ったときの基準）

優先順位は基本この順番。

1. 数字（利益インパクト）
2. 再現性（仕組みにできるか）
3. スピード（すぐ試せるか）
4. 人（誰がやるか）
5. リスク

**「利益 × 再現性 × スピード」で判断する傾向が強い。**

---

## 優先順位の付け方

**やることが多いときは：**
1. 売上・利益に直結するもの
2. 構造を変えるもの（仕組み化）
3. 将来の武器になるもの

**捨てる基準：**
- 売上に影響が薄い
- 属人的で再現性がない
- 考える時間の割に成果が小さい

**「インパクト × 再現性」で優先順位を決める。**

---

## 見落としがちな観点

過去の経験から、注意すべき点。

1. **現場の処理能力** … 施策は良くても現場が回らないケース
2. **人材の理解レベル** … 自分は理解できるが組織が理解できない
3. **優先順位の共有不足** … 自分は明確でも組織に伝わっていない
4. **実装コスト** … 仕組みが正しくても運用が重すぎる

→ AI・参謀には「ここも見て」と伝えておく。

---

## よく使う問い（思考トリガー）

意思決定時に自分へ投げる問い。

- これは再現できるか？
- スケールするか？
- 数字はどれくらい動く？
- 誰でもできる形にできる？
- テストできるか？
- 今やる意味はある？

**特に「再現できるか？」が強い。**

---

## 妥協しない線・譲れないこと

1. **数字の誤魔化しはNG** … 数字は必ず検証する
2. **顧客対応の品質** … 信用は事業の資産
3. **再現性のない成功** … 属人化した成功は成功として扱わない
4. **スピード** … 考えるだけで実行しない状態はNG

## 補足：CEOの得意・不得意

- 継続したり、細かいマネジメントは苦手。
- その分、戦略・判断・仕組み化・外部に集中する設計にする。
