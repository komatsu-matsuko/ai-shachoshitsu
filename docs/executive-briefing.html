経営レイヤー・やること・報告フロー（図解）

Bremen株式会社（Safari / 麺昇 神の手 / Studio13）向け。
目的は AI＝思考／CEO＝判断／幹部＝実行 を徹底して、判断を速く・再現性高く回すこと。 経営判断の基準：利益 × 再現性 × スピード

Insight → Recommendation → Execution Plan で出す。

  • 水野：数字の分解・KPI
  • 真田：戦略案（最大3案）
  • 相沢：実行計画（誰が/いつまで）

CEOの役割（判断）

運営管理はしない。判断の質を上げる。

  • まず数字 → 再現性 → 小さく試す
  • 組織で回るか（属人化NG）
  • 実行してデータを取る
参照 08_ceo/ceo-thinking-framework.md

  • 岡：店舗・オペ

  • 結城：管理・運用

やることの見える化（クエストボード運用）

「明日やること」を1画面にして、実行速度を上げる

🎯 明日やること（クエスト）

1つの最重要タスクを「メインクエスト」に置き、サブクエストをチェックで潰す。

  • まず「勝利条件（Win）」を1行で決める
  • 担当を「主担当（進行）」と「実装/実行」に分けて固定する
  • ブロッカーは「ボス」として別枠に出す（放置しない）
  • 夜にAI経営会議で「実行計画」まで落とす
参照：company-memory/10_dashboard/明日やること.md

🧩 社長室が「幹部に与える能力」

幹部が強くなるのは「才能」より「型」。このページの型を渡す。

  • 報告は ①先週やったこと ②問題 ③次にやること で統一
  • 週次MTGは ①先週レビュー ②KPI ③問題 ④次アクション の順で固定
  • 施策は 利益 × 再現性 × スピード で優先順位を付ける
  • 指示は必ず やること｜担当｜期限 で出す（曖昧指示禁止）
参照：運用仕様書.md / 08_ceo/ceo-thinking-framework.md
