# フォルダの中身を埋めるガイド

AI社長室の精度は、**company-memory にどれだけ会社の情報が入っているか**で決まります。  
どこから埋めればよいか、順番とポイントをまとめました。

---

## 推奨する埋める順番

| 順番 | フォルダ | 理由 |
|------|----------|------|
| **1** | **08_ceo** | CEOの考え方・判断基準が入っていると、AIの答えが一気に「社長に近く」なる。最優先。 |
| **2** | **01_company** | 組織・事業概要が分かると、誰に何を任せるか・どの事業の話かが明確になる。 |
| **3** | **03_business** | 買取・飲食・デザインの責任者・課題・方針が分かると、戦略案や実行計画が具体化する。 |
| **4** | **04_kpi** | 数字の定義と目標が分かると、水野の分析・ブリーフの質が上がる。 |
| 5 | 02_strategy | 中期・成長・市場ポジション。戦略議論の土台。 |
| 6 | 05_projects | 今動いているプロジェクト。実行計画とタスクの整合が取りやすくなる。 |
| 7 | 06_operations | 業務フロー・ツール・自動化。工藤の提案が現実的になる。 |
| 8 | 07_knowledge | 教訓・ベストプラクティス・市場知見。神谷のリスク指摘が効く。 |
| 9 | 09_meetings | 週次MTG・月次レビューのログ。会議のたびに1ファイル追加。 |
| 10 | 10_dashboard | 社長ダッシュボードの元データ。週1回などで更新。 |

**まず 08_ceo → 01_company → 03_business → 04_kpi を埋めると、すぐに「使える」体感が得られます。**

---

## 01_company と 03_business に必要な情報（チェックリスト）

### 01_company（会社の基本）

**company_profile.md に欲しい情報**
- 会社の正式名称（または通称）
- 設立年（または創業のイメージ）
- 本社・拠点（都道府県やエリア程度でOK）
- 事業内容を一文で（例：「買取・飲食・デザインの3事業」）

**organization.md**  
→ すでに骨子あり。役職・担当者名の誤りや補足があれば追記。

**business_overview.md**  
→ 利益構造・営業フロー・各事業の流れはすでに骨子あり。具体的な商品名・サービス名・顧客層を足すとAIの回答がより具体化する。

---

### 03_business（各事業の定義）

**各事業ファイル（buying / restaurant / design）に、共通で欲しい情報**

| 項目 | 必要な情報の例 |
|------|----------------|
| **事業名** | 社内で呼んでいる名前（例：買取事業、〇〇店、デザイン事業） |
| **責任者** | 担当者名（すでに笹川・岡・吉浦と分かっている場合は「笹川」などでOK） |
| **利益構造** | どうやって稼ぐか（例：催事で買い取り→転売で粗利、客数×客単価−原価−人件費、案件単価×件数−制作コスト） |
| **主要KPI** | ファイルに列挙済み。目標値や「いま見ている数字」があれば追記するとよい |
| **現在の課題** | 今抱えている問題・ボトルネック（1〜3個でOK） |
| **今後の方針** | これからやりたいこと・伸ばしたい方向（短くてよい） |

**書くときのコツ**
- 完璧な文章でなくてよい。箇条書き・メモでOK。
- 「課題」は、AIが戦略案や実行計画を出すときに「ここを解消する」前提で話せるので重要。
- 「今後の方針」があると、真田の戦略案がぶれにくくなる。

## フォルダ別：何を書くか

### 08_ceo（最優先）

| ファイル | 書くこと |
|----------|----------|
| **ceo-thinking-framework.md** | 思考の型・意思決定の流れ・時間配分（すでに骨子あり。追記でOK） |
| decision_style.md | どんなときに即決／熟考するか、好む情報の形式 |
| thinking_rules.md | 判断時のルール（例：数字は必ず見る、など） |
| priorities.md | 今期・今年の優先順位（すでに骨子あり） |
| principles.md | 経営で譲れないこと・信念 |

### 01_company

| ファイル | 書くこと |
|----------|----------|
| company_profile.md | 会社名・設立・本社・事業内容の一文 |
| organization.md | 役職と担当者（すでに骨子あり。名前・役割の補足でOK） |
| business_overview.md | 利益構造・営業フロー・各事業の流れ（すでに骨子あり） |

### 03_business

| ファイル | 書くこと |
|----------|----------|
| buying_business.md | 事業名・責任者・利益の仕組み・主要KPI・今の課題・今後の方針 |
| restaurant_business.md | 同上 |
| design_business.md | 同上 |

### 04_kpi

| ファイル | 書くこと |
|----------|----------|
| kpi_definition.md | 全社で見るKPIの名前・定義・目標値（または「04_kpi 内の各事業ファイルに記載」でOK） |
| buying_kpi.md | 催事件数・売上・粗利率・成約率・平均単価の目標 or 直近実績 |
| restaurant_kpi.md | 売上・客数・客単価・原価率・人件費率 |
| design_kpi.md | 案件数・売上・平均単価・利益率・制作時間 |

### 02_strategy

| ファイル | 書くこと |
|----------|----------|
| midterm_strategy.md | 中期（2〜3年）でどこを目指すか・方針 |
| growth_strategy.md | どう伸ばすか・スケールの考え方 |
| market_position.md | 市場での立ち位置・競合・差別化 |

### 05_projects

| ファイル | 書くこと |
|----------|----------|
| project_list.md | プロジェクト名・目的・ステータスの一覧 |
| active_projects.md | 今動いているものの詳細（担当・期限・次のマイルストーン） |

### 06_operations

| ファイル | 書くこと |
|----------|----------|
| workflow.md | 主要業務の流れ（例：受注〜納品、週次MTGの流れ） |
| tools.md | 使っているツール（Slack, Notion, Google, BI など） |
| automation.md | すでに自動化していること・これからやりたいこと |

### 07_knowledge

| ファイル | 書くこと |
|----------|----------|
| lessons.md | 過去の失敗・成功から得た教訓（箇条書きでOK） |
| best_practices.md | 社内でうまくいっているやり方 |
| market_insights.md | 業界・競合・市場のメモ |

### 09_meetings

- **weekly/** … 週次MTGのたびに、日付のファイルを1つ追加（例：`2026-03-15_weekly_mtg.md`）。  
  中身：参加者・各事業の進捗・KPI・問題・次のアクション・決定・タスク表。
- **monthly/** … 月末に月次レビューを1ファイル。  
  中身：月次総括・事業別レビュー・重点改善。

### 10_dashboard

| ファイル | 書くこと |
|----------|----------|
| ceo_dashboard.md | 今の「今日の要点・全社サマリー・重点・アラート・社長メモ」のスナップショット |
| weekly_summary.md | 今週のハイライト・事業別サマリ・来週の重点（週1更新がおすすめ） |
| alerts.md | 役員アラート・遅延案件・要判断事項（随時追加） |

---

## 書き方のコツ

- **完璧でなくてOK** … 箇条書き・メモ程度でよい。あとから足していける。
- **08_ceo を厚くする** … ここが充実していると、AIのトーンと判断が社長に近くなる。
- **数字は 04_kpi に集約** … 目標値・直近実績を書いておくと、水野の分析が使える。
- **週次MTG後に 09_meetings に1ファイル** … 続けると、過去の文脈をAIが参照できる。

---

## 次のアクション

1. **08_ceo** から、ceo-thinking-framework.md と principles.md に追記する。
2. **01_company** の company_profile.md に会社名・設立・本社などを入れる。
3. **03_business** の3ファイルに、各事業の「責任者・課題・方針」を書く。
4. **04_kpi** に、今わかっている数字・目標を書く。

ここまで埋まれば、「おはよう」「この問題を分析して」「経営会議」の回答がかなり具体化します。  
特定のファイルの中身を一緒に書きたいときは、「〇〇のファイルを埋めたい」と指示してもらえれば、項目案や文例を出します。
